guh.nvim                                                            *guh*

Work with GitHub PRs in Neovim. Wraps the GitHub `gh` CLI tool and provides
a very minimalist yet effective workflow.

Tenets:
- Never lose data. (TODO: use a real file for edit_comment, available on crash / failed post.)
- Load data eagerly (Kool-Aid man "OH YEAH!"), and never request it again
  until user explicitly invokes "refresh" (R).
- Avoid messing with the workspace at all. Try to work with buffers and PR
  data without requiring a checkout.
- All mappings work in all Guh buffers ("guh://…").
- Use buffers as the "storage" system. Buffers are created with
  'bufhidden'=hide, user config must not change that.

==============================================================================
INSTALLATION                                                     *guh-install*

Requirements:
- Nvim 0.13+
- "gh" (GitHub CLI): https://cli.github.com/
- (Optional) For working with Git, use a plugin such as: https://github.com/tpope/vim-fugitive
- (Optional) For highlighting diffs, use a plugin such as: https://github.com/barrettruth/diffs.nvim
- (Optional) For improved Markdown rendering, use a plugin such as: https://github.com/MeanderingProgrammer/render-markdown.nvim

Install: >lua
  vim.pack.add{ 'https://github.com/justinmk/guh.nvim' }
<

WORKFLOW                                                           *guh-usage*

Guh does not provide a setup() function. Instead it defines a plugin/guh.lua
file which Nvim automatically runs at startup. |lua-plugin-init|

To review a PR without doing a checkout in your repo:

- Run `:Guh` to see the current repo status (commits/PRs/issues), or the
  global "guh://status" (notifications) page if no repo can be guessed from
  the current buffer or directory.
  - Hit "-" to go "up" to your notifications ("guh://status").
  - Hit <Enter> or `CTRL-W<Enter>` on a commit/PR/issue to view it.
- Run `:Guh [target]` to view a specific PR/issue. |guh-commands|
  - Hit `R` to refresh the view.
- All "guh://…" buffers have the same basic mappings. |guh-mappings|
- From the PR view, hit `dd` to see the diff. |guh-diff|
- PR comments are loaded:
  - in a 'scrollbind' split window next to the diff window (if the
    scrollbind gets out of sync, try |:syncbind|).
  - as diagnostics on the diff buffer (severity INFO, source "guh.nvim").
    Navigate via |]d|/|[d|; peek with |CTRL-W_d|.
  - in the quickfix list ("Guh comments"). |]q| |[q| |:copen|
- Hit `cc` or `:GuhComment` to comment on a diff line.

                                                                 *guh-confirm*
To confirm an action (such as "merge", "post comment", "approve", etc.) which
accepts a (potentially optional) text input, Guh presents a "guh://…/edit".
- To confirm the action, just save-and-close the buffer: `:wq`, `ZZ`, etc.
- To cancel/abort the action, close-without-write (`:q!` or `ZQ`). Or if the
  action requires non-empty content, writing an empty buffer will also abort.

Keymaps                                                           *guh-config*

Guh defines global `<Plug>(guh-…)` keys at startup. You can map anything to
those |<Plug>| keys to "override" the default Guh keymaps. Each `guh://`
buffer sets buffer-local default mappings unless you defined your own.

Example: `<Plug>(guh-merge)` is mapped to "cM" by default, but if you define
your own mapping, Guh will not map "cM": >vim
  nnoremap <leader>m <Plug>(guh-merge)

If you want to skip the menu for an action, just map `[count]<Plug>(guh-…)`,
where [count] is the menu-item you want to select.

Example: If you want "D" to toggle Draft/Ready, map it to menu-item 4: >vim
  nnoremap D 4<Plug>(guh-review)
<
Example: Try mapping ":Guh ." globally, then you can use it from anywhere,
even non-Guh buffers:
>vim
  nnoremap Ug <cmd>Guh .<cr>
<
                                                                *guh-mappings*
Default mappings (local to `guh://` buffers):

BUFFER-RELATIVE VIEW ACTIONS

-       Go "up" to the parent of the current thing.
        - From a prdiff/, prcomments/, prlogs/, file/ buffer: opens the pr/ buffer.
        - From a pr/ or issue/ buffer: opens the repo page ("guh://owner/repo").
        - From the repo page: opens "guh://status".
        <Plug>(guh-up)

R       Refresh the current thing. In a PR-related buffer, refreshes all
        objects related to that PR (except immutable objects such as prlogs/
        and commit/).
        <Plug>(guh-refresh)

dd      View PR diff + comments. |guh-diff|
        <Plug>(guh-diff)

dl      View the most-recent CI logs for all job-names in the CI matrix.
        <Plug>(guh-logs)

]f      View the next "thing".
        - In a CI logs buffer: next CI job.
        - In other PR-related buffers: next commit (from the most-recent push).
        <Plug>(guh-next)

[f      View the previous "thing" (see ]f above).
        <Plug>(guh-prev)

gX      Open the current view on github.com in your web browser.
        <Plug>(guh-web)

g?      View this help section.
        <Plug>(guh-help)

BUFFER-RELATIVE UPDATE ACTIONS

cC      Comment on the PR/issue (top-level). |guh-comment|
        <Plug>(guh-comment-top)

cI      Rerun CI jobs. Opens a picker (or use `[count]cI` to skip):
        1. Current job (from a CI logs buffer)
        2. Failed jobs

        <Plug>(guh-ci)

cM      Merge PR. |guh-merge|
        <Plug>(guh-merge)

cR      Review PR (Approve / Comment / Request-changes / Toggle draft).
        <Plug>(guh-review)

cV      Mark the current PR/issue as "Read". Depends on your "guh://status"
        notifications. Note:
        - Guh never marks anything as "Read" unless you invoke this.
        - Cannot be undone; GitHub has no "Mark as Unread" API.
        <Plug>(guh-notif-read)

c:      Edit title/body/labels/milestone/… via `gh` in a :terminal.
        <Plug>(guh-edit)

CURSOR-RELATIVE ACTIONS

<Enter>
        Invoke |:Guh| on the |WORD| at cursor.
        <Plug>(guh-open)

CTRL-W<Enter>
        Invoke |:Guh| on the |WORD| at cursor, in a split window.
        <Plug>(guh-open-split)

cc
{Visual}c
        Comment on a diff-line. In a `guh://…/prcomments/` buffer, edits the
        comment at cursor. |guh-comment|
        <Plug>(guh-comment)

cr
        Reply-to/Resolve the thread at cursor, in a `guh://…/prdiff/` or
        `guh://…/prcomments/` buffer. |guh-comment|
        <Plug>(guh-thread)

cv      Toggle "Viewed" state of the diff-file at cursor. Also works on
        OUTDATED/OUTSIDE diff-files, but only locally: "R" (refresh) will show
        those files again if they have Unresolved threads.
        <Plug>(guh-viewed)

gf      In a prdiff/ buffer, view the full contents of the diff-file at
        cursor.
        <Plug>(guh-file)

]]      In a prdiff/ buffer, jump [count] sections forward, where each
        "section" is a "diff --git" heading.

[[      Like ]], except backwards.

View PR diff                                                        *guh-diff*

`dd` on a PR view opens the diff + comments in 'scrollbind' split windows.
Comments are also provided as diagnostics (and loaded into the quickfix list).

- Only UNRESOLVED comments are shown.
- Comments on VIEWED files are never shown, even if Unresolved.
- Unresolved comments on OUTDATED code (i.e. code from an old push), are
  anchored to pseudo-diff sections above the "current" PR diff, with
  pseudo-filename `outdated-<id>:<file>`.
- Unresolved comments on code OUTSIDE of the PR diff, are similarly anchored
  to pseudo-filename `outside-<id>:<file>`.
- The current PR diff follows the above sections.

PR diff:

- Lists diff-hunks in this order:
  1. Outdated (if any)
  2. Outside (if any)
  3. Current
- Shows VIEWED files as a single line (diff/comments not loaded): >
  (viewed) path/to/file
<
- The diff overlay shows: >
  PR diff (Viewed files: 3/12, Resolved comments: 4/20).
<
- Refresh ("R") to update after marking/unmarking a file as VIEWED.

PR comments:

- in a 'scrollbind' split window next to the diff window (if the scrollbind
  gets out of sync, try |:syncbind|).
- when multiple comments anchor to the same diff line, every comment's
  heading row is guaranteed; the remaining vertical budget is filled with
  comment bodies in-order. If a comment can't fit, it shows `(truncated)`. The
  full comment is always accessible via the diagnostic peek (`<C-w>d`) or the
  quickfix list.
- as diagnostics on the diff buffer (severity INFO, source "guh.nvim"):
  - Navigate using `vim.diagnostic.jump` (|]d| |[d|).
  - See comment details using `<C-w>d` |CTRL-W_d-default|.
- in the quickfix list ("Guh comments") via |vim.diagnostic.setqflist()|.
  Use |]q|/|[q| or |:copen| to browse.

------------------------------------------------------------------------------
Comment on issues, PRs, and diffs                                *guh-comment*

`cc` on a PR diff-line, or on an existing comment, opens an editable
`guh://…/comment/<id>` buffer. Write-and-close to post the comment. |guh-confirm|

`cr` on a comment (thread), presents a menu:
1. Reply
2. Resolve

See |:GuhComment|.

------------------------------------------------------------------------------
Merging PRs                                                        *guh-merge*

`cM` on a PR view opens a picker:

1. squash
2. merge
3. rebase
4. squash --admin (bypass branch protections)
5. merge --admin (bypass branch protections)
6. rebase --admin (bypass branch protections)

`[count]cM` skips the picker (e.g. `1cM` = squash, `5cM` = merge --admin).

For "squash" and "merge", Guh presents a `guh://…/merge/<id>` buffer,
prefilled with a title and body.
- for "merge": body = PR description
- for "squash": body = commit-body, or PR description if commit-body is empty

Write-and-close to confirm the "merge" action. |guh-confirm|

------------------------------------------------------------------------------
Review PRs: approve, request changes, or reject                   *guh-review*

`cR` on a PR view opens a picker:

1. Approve
2. Comment
3. Request-changes
4. Draft/Undraft: toggles the PR "ready for review" state.

`[count]cR` skips the picker (`1cR` = approve, `4cR` = toggle draft, …).

To confirm the "review" action, Guh presents a `guh://…/review/<id>` buffer.
Write-and-close to confirm the "review" action. |guh-confirm|

COMMANDS                                                        *guh-commands*

:Guh [target]                                                           *:Guh*

Views or reloads the given [target], or shows a status if [target] is not
given (or is a repo). Accepts |<mods>|: >

    :vertical Guh …
    :horizontal Guh …
    :tab Guh …

If |<mods>| is not given, the existing window for [target] (if any) is
focused. PR data is eagerly loaded (Kool-Aid man "OH YEAH!").

Without [target]: shows the current repo ("guh://<owner>/<repo>"), resolved
from the current buffer or directory.
- The 10 most-recent commits (on the default branch).
- Open PRs and issues (recently-updated).
- Your `gh pr status` for the repo.
- Hit <Enter> or `CTRL-W<Enter>` to view any commit/PR/issue at cursor.
- Hit "-" to go up to "guh://status" (your notifications). |guh-mappings|
- If a repo can't be resolved, then :Guh shows your notifications
  ("guh://status").

With [target]: views or refreshes the PR, issue, commit, repo, or global status. >vim
    " PR or issue number (current repo)
    :Guh 35951
    " Commit SHA (current repo)
    :Guh a1b2c3d
    " GitHub URL
    :Guh https://github.com/neovim/neovim/pull/35951
    " GitHub commit URL
    :Guh https://github.com/neovim/neovim/commit/a1b2c3d
    " Repo URL
    :Guh https://github.com/neovim/neovim
    " Slug ("owner/repo#123" or "owner/repo")
    :Guh neovim/neovim#35951
    :Guh #35951
    :Guh neovim/neovim
    " guh:// URI
    :Guh guh://neovim/neovim/pr/35951
    :Guh guh://neovim/neovim
    " WORD at cursor
    :Guh .

When [target] is a number, "#N", or ".", the repo is resolved relative to the
current buffer, falling back to the window-local |current-directory|.

For a PR target, `:Guh` loads the pr/ + prdiff/ + prcomments/ views.
The pr/ view becomes the |alternate-file| of the prdiff/ view (so `CTRL-^`
returns to it).

Commits are fetched via the GitHub API (no local checkout required) and
shown as a |ft-gitcommit-plugin| buffer. <Enter> on a SHA in any `guh://`
buffer, opens that commit.

Related gh commands: >
    gh status
    gh pr status
    gh pr view 42
    gh issue view 42
    gh api repos/OWNER/REPO/commits/SHA
<

:[range]GuhComment                                               *:GuhComment*

Creates, updates, or deletes a comment. >

    :.GuhComment
    :'<,'>GuhComment

Use range "%" to create a top-level comment on the current PR/issue: >

    :%GuhComment

Use bang "!" with a single-line [range] to delete a comment: >

    :.GuhComment!
<

HIGHLIGHTS                                                    *guh-highlights*

Guh defines these highlight groups (with `default = true`, so a colorscheme
that defines them wins). Link or `:hi` them to customize.

GuhHeading
    Comment-thread title, and various other headings.

GuhWarning
    Hints such as `(outdated)`, `(outside)`, `(truncated)`, `(viewed)`.
    Pending CI checks.
    "Unread" status.

DEVELOPMENT                                                          *guh-dev*

Enable debug logging: >lua

    vim.g.guh_debug = 'debug'   -- Plugin-side logs + stderr capture
    vim.g.guh_debug = 'trace'   -- + GH_DEBUG=api request/response tracing

View the logs: >

    :log guh

Run the tests: >

    NEOVIM_PATH='/path/to/neovim/' make test

Run specific tests: >

    NEOVIM_PATH='/path/to/neovim/' make test TEST_FILTER=load_comments

------------------------------------------------------------------------------
ARCHITECTURE

- "Reload" internally is designed to work by "clearing b:guh" (i.e. call
  state.invalidate()). Don't pass `force` parameters around in a bespoke
  fashion. And as a result, guh:// buffers are by default _never_
  reloaded _unless_ b:guh was cleared.
